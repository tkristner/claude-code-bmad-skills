# Documentation Quality Checklist

Use this checklist to validate documentation before publishing.

---

## Content Quality

### Accuracy

- [ ] Information is factually correct
- [ ] Code examples work as written
- [ ] Screenshots match current UI
- [ ] Version numbers are correct
- [ ] Links point to correct destinations

### Completeness

- [ ] All necessary topics are covered
- [ ] No "TODO" or placeholder content
- [ ] Prerequisites are listed
- [ ] Error handling is documented
- [ ] Edge cases are addressed

### Currency

- [ ] Content reflects current version
- [ ] Deprecated features are marked
- [ ] No references to removed features
- [ ] Dates are accurate

---

## Structure Quality

### Organization

- [ ] Logical flow from start to finish
- [ ] Related topics are grouped
- [ ] Complex topics are broken into sections
- [ ] Table of contents for long documents

### Navigation

- [ ] Clear heading hierarchy (H1 > H2 > H3)
- [ ] No skipped heading levels
- [ ] Cross-references to related content
- [ ] Index or search available (if applicable)

### Formatting

- [ ] Consistent heading styles
- [ ] Proper use of lists
- [ ] Code blocks have language specified
- [ ] Tables are properly formatted

---

## Readability

### Audience Appropriate

- [ ] Written for target audience
- [ ] Technical level matches readers
- [ ] Jargon is defined or avoided
- [ ] Assumptions are stated

### Clarity

- [ ] Active voice used predominantly
- [ ] Sentences are concise (under 25 words)
- [ ] Paragraphs focus on one idea
- [ ] Examples illustrate concepts

### Scannability

- [ ] Headings describe content
- [ ] Key information is highlighted
- [ ] Lists break up long text
- [ ] Important warnings stand out

---

## Technical Accuracy

### Code Examples

- [ ] Code compiles/runs without errors
- [ ] Examples are minimal but complete
- [ ] Output matches what's shown
- [ ] Dependencies are specified

### Commands

- [ ] Commands work when copy-pasted
- [ ] Platform differences noted
- [ ] Required permissions mentioned
- [ ] Environment requirements stated

### API Documentation

- [ ] All endpoints documented
- [ ] Parameters are accurate
- [ ] Response formats are correct
- [ ] Error codes are listed

---

## Accessibility

### Universal Access

- [ ] Images have alt text
- [ ] Links are descriptive
- [ ] Color is not only indicator
- [ ] Content works without images

### Internationalization

- [ ] No culture-specific references
- [ ] Dates in ISO format or clarified
- [ ] Units of measurement clarified
- [ ] Simple language for translation

---

## Maintenance

### Sustainability

- [ ] Easy to update
- [ ] No hardcoded values that change
- [ ] Single source of truth
- [ ] Clear ownership

### Version Control

- [ ] Changes tracked in version control
- [ ] Meaningful commit messages
- [ ] Review process in place

---

## Final Review

### Before Publishing

- [ ] Spell check completed
- [ ] Grammar check completed
- [ ] All links verified
- [ ] Renders correctly in target format
- [ ] Reviewed by someone else
